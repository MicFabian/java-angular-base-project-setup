# Architecture Decision Records

This folder stores Architecture Decision Records (ADRs) for the project.

## Naming

- Use `NNNN-short-kebab-title.md`.
- Start at `0001`.
- Keep the number stable once published.

Examples:

- `0001-repository-structure.md`
- `0002-backend-clean-architecture.md`

## Status Values

- `Proposed`
- `Accepted`
- `Superseded`
- `Deprecated`

## ADR Lifecycle

1. Create ADR in `Proposed` when a meaningful technical decision is being made.
2. Mark as `Accepted` when implementation direction is confirmed.
3. If replaced, keep old ADR and mark it `Superseded` with a link to the new ADR.

Use [0000-template.md](./0000-template.md) as the source template.
